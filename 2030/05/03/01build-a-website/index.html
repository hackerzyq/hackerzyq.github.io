问题汇总

Error: EACCES: permission denied 问题

解决方案：所有终端命令前 加上 sudo （管理员权限执行）

输入 sudo hexo deploy 还是出现 Permission Denied 问题

在输入 sudo hexo deploy 并出现了经典的 本地没有ssh 对应github 上面的 ssh的问题。也就是Permission Denied。对于该问题，为什么会出现permission denied，很明显就是没有ssh。这也是linux权限的问题。首先要使用hexo 必须使用sudo获取linux下的最高权限 ，sudo目录就是linux下的root，当你用 ssh-keygen -t rsa -C “xxx@xxx.com” 添加ssh的时候都是在自己的账户下添加的，而没有到root文件夹下去添加，所以sudo hexo deploy就会出现ssh不存在的permission denied问题。
解决方案：直接在root下添加ssh就可以解决这个问题。我们来一步一步的解决如何添加ssh到linux root目录下面。

1.在linux的root目录下创建ssh，加上sudo。

2.然后利用cat命令打开在root目录下的ssh。

3.在github上面进行ssh的粘贴

域名购买并使用

前面七步已经搭建好了个人网站，以git作为服务器，ip地址为 xxx.github.io
当然，大家都会希望有一个自己的域名。

1.查找自建网站的 DNS 地址

打开终端，ping自己的网站 ip 地址，查看 DNS 。

2.域名购买

进入万网，选择域名并且购买。注册并通过支付宝购买，便捷。

3.解析域名

进入个人账户的管理控制台，选择 域名与网站 - 域名，在当前页面找到自己购买的域名，点击后面的解析，找到记录类型为A的信息栏，修改其记录值为刚刚在终端所见的 DNS 地址。保存。此时在网址打开新买的域名还是出现 404错误，没有问题。

4.创建 CNAME

用终端打开网站根目录文件夹，在 source 文件夹下创建 CNAME 文件，文件里加上新买的域名地址。
